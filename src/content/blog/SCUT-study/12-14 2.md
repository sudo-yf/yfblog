保留发音好听、寓意精准且符合“悬浮条+视觉模型+中枢控制”**特征的名字。

1. Mentat (门塔特)
出处： 科幻巨著《沙丘》 (Dune)
词源： 拉丁语词根 ment- (Mind/思维)。
为什么完美适配：
    人设： 在《沙丘》中，Mentat 是“晶算师”，是经过训练能像超级计算机一样进行逻辑分析和计算的人类。
    产品映射： 你的 Agent 正是如此——它有着人类的视觉（UI-TARS 像人眼一样看 GUI），同时有着计算机的逻辑（Code/Function Call）。它是“人机结合”的终极形态。
    GitHub 风格： mentat 或 mentat-ui。听起来非常硬核、智能。

2. Limen (阈限)
出处： 心理学/人类学概念，源自拉丁语。
词源： 拉丁语 limen，意为**“门槛”或“入口”。
为什么完美适配：
    形态映射： 那个悬浮的搜索栏，物理上就是横亘在“用户意图”和“复杂系统”之间的一道门槛。
    功能映射： 用户跨过这道门槛（输入指令），Agent 就在另一侧调用 MCP 和 Function Call 来改变世界。
    GitHub 风格： limen。短促、冷门、极具高级感。

3. Nemo (尼摩)
出处： 凡尔纳《海底两万里》 (Twenty Thousand Leagues Under the Sea)
词源： 拉丁语 nemo，意为**“没有人”** (No one)。
为什么完美适配：
    视觉隐喻： 悬浮窗平时是静止的、极简的，像一艘潜水艇悬浮在深海。
    无形的服务： 就像尼摩船长驾驶鹦鹉螺号在海底解决一切，你的 Agent 也是一个**“隐形”的操控者**。它不需要复杂的界面，它只是默默地帮你在后台把 GUI 和 Code 的活儿干了。
    GitHub 风格： nemo-agent。亲切，且带有一种“幕后黑手”的掌控力。

4. Sutra (经纬)
出处： 东方哲学/梵语经典。
词源： 梵语 sūtra，原意为**“线”** (Thread/String)。
为什么完美适配：
    技术映射： 这一点非常打动开发者。Function Call 本质上就是把不同的服务串联起来；程序执行也是一种“线程”。
    操作映射： 你的 Agent 像一根针线，穿梭于不同的 GUI 窗口（布料）之间，把破碎的工作流缝合在一起。
    GitHub 风格： sutra 或 sutra-core。听起来逻辑严密，结构稳固。

5. Argus (阿戈斯)
出处： 希腊神话。
背景： 拥有一百只眼睛的巨人，赫拉的守望者。
为什么完美适配：
    核心致敬： 直接致敬 UI-TARS 的视觉能力。
    功能映射： 这个 Agent 能够“看穿”屏幕上的每一个像素，监控每一个 UI 变化，就像拥有百眼的阿戈斯一样，没有任何细节能逃过它的捕捉。
    GitHub 风格： argus 或 argus-bar。非常有力量感，强调“视觉识别”这一核心卖点。

⚡️ 最终决策指南

如果你想强调**“它是代替你思考的超脑”** \rightarrow 选 Mentat (极客首选)
如果你想强调**“极简的悬浮交互形态”** \rightarrow 选 Limen (设计感首选)
如果你想强调**“强大的视觉识别能力”** \rightarrow 选 Argus (技术流首选)
如果你想强调**“幕后默默执行任务”** \rightarrow 选 Nemo (大众亲和力)
